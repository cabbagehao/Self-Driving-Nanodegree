车辆会有3种行为：
1. 道路畅通，保持车道
	如果当前车道前方车辆在探测距离30m外，则以最大限速在当前车道行驶。
2. 遭遇拥堵，无法变道时速度与前车保持一致。
	如果当前车道前方车辆在30m内，且相邻车道空隙不够，则减速与前面车辆相同速度。
3. 遭遇拥堵，可以变道时立即变道。
	如果当前车道前方车辆在30m内，且相邻车道有足够空隙，则立即变道。


如何生成的轨迹：
	1. 每次会预测50个点作为车辆运行的轨迹
	2. 这50个点由2部分组成：
		1. 上一次生成的previous_path_x的unprocessed points。
		2. 新生成的轨迹。
	3. 新生成的轨迹使用了spline库去拟合一条曲线。  spline使用的points由2部分组成：
		1. 上一次计算的previous_path的后最2个点
		2. 在计算出的车道上取3个点。 这3个点的d值为该车道的中心，s值分别为当前车辆s的前面45 55 65m处。
		3. 将这些点变换到map 全局坐标系。
		这样，既能保证新生成的轨迹与以前轨迹的拟合，也能保证变道后不会有太大的jerk。

如何变道：
	1. 计算车辆当前车道， 和相邻2条车道（可能只有1条，比如车辆在路的最右边）。
	2. 计算相邻车道的前后最近的2辆车是否有足够间距。
	3. 如果有足够间距，则立即变道。  如果没有间距，则减速与前方车辆一样的速度，并等待机会变道。
